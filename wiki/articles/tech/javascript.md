# JavaScript

JavaScript is the core language [[Hendrix]] builds in. It runs everything from the front-end of [[Duodode]] products to the [[Node.js]] runtime underneath them.

## Why it matters to him

One language across the stack keeps his thinking and tooling tight. [[Next.js]], [[Supabase]] client code, [[n8n]] custom logic, and front-end interactivity all share the same base. That compounds.

## How he uses it

- Building interactive UI inside [[Next.js]] and [[Frontend Frameworks]]
- Writing logic that bridges [[Frontend Frameworks|front-end]] and [[Backend]] work
- Automating tasks in [[Node.js]]
- Pairing with [[Claude Code]] and [[Codex]] for [[AI-Assisted Development]]
- Writing custom steps inside [[n8n]] flows

## Where it connects

- Runs the browser layer on top of [[HTML]] and [[CSS]]
- Runs the server through [[Node.js]]
- Powers motion work with [[GSAP]] and [[Framer Motion]]
- Drives interactions inside [[Rive]]-enhanced UIs

## How it fits his approach

JavaScript is leverage per unit of time. A builder who knows it well can move across design, product, backend, and automation without trading tools constantly. That matches Hendrix's preference for [[Leverage|compounding tools]] and [[Build and Ship|fast shipping]].

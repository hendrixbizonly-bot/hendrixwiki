# CSS

CSS is where my design taste meets the browser. It is how my [[Figma]] work becomes real products. For someone with a strong [[Design Philosophy|design philosophy]], CSS is not a detail. It is the final mile.

## Why it matters to me

A design is only as good as its implementation. [[Intentional Design|intentional]], [[Timeless Design|timeless]], and [[Consistent Design|consistent]] only hold up if the CSS actually delivers them. Strong CSS keeps work from degrading between file and browser.

## How I use it

- Implementing [[Branding]] systems in real products
- Building UI that matches the precision of [[Figma]] designs
- Supporting motion work from [[GSAP]] and [[Framer Motion]]
- Producing responsive, resilient layouts across devices

## Where it connects

- Built on top of [[HTML]]
- Often written through tooling inside [[Next.js]] and [[Frontend Frameworks]]
- Coordinated with motion libraries for [[Motion Design]] and [[Interaction Design]]

## How it fits my standard

CSS is one of the clearest surfaces where taste shows. I use it to push work past the generic defaults of any framework into something that feels specific, coherent, and my.

# GitHub

GitHub is where [[Hendrix]] stores and manages code. Every [[Duodode]] project, every experiment, and every build he runs through [[VS Code]] lives behind it. It is the backbone of his technical workflow.

## What it is

A hosted platform for git repositories, with collaboration, deployment integrations, and issue tracking built on top.

## How Hendrix uses it

- Version control for all project code
- Deployment pipelines into [[Vercel]]
- Collaboration on client and internal work
- A clean history of changes across projects
- A trusted home for work built with [[AI-Assisted Development]]

## Why it fits his workflow

Every serious modern stack assumes GitHub. Tying the rest of his tools to it ([[VS Code]], [[Vercel]], [[Next.js]]) means everything plugs in without extra plumbing. That matches his preference for tools that create [[Leverage|leverage]] and remove friction.

## What it unlocks for him

- Safe, reversible building
- Instant deployment through connected services
- A shared surface for client projects
- A track record of work that compounds over time

# Node.js

Node.js is the JavaScript runtime underneath most of [[Hendrix]]'s technical work. It is what powers [[Next.js]] on the server, scripts he runs locally, and tooling across his stack.

## What it is

A JavaScript runtime that lets JS run outside the browser. The default engine for modern web and product tooling.

## How Hendrix uses it

- Running [[Next.js]] apps in development and production
- Executing build tools, scripts, and CLI utilities
- Supporting API work tested through [[Postman]]
- Providing the runtime underneath deployments to [[Vercel]]
- Acting as the default language context for anything involving [[JavaScript]]

## Why it fits his workflow

Most of his stack assumes Node. Sticking close to it means fewer context switches and a tighter loop between front-end and back-end work. That matches his bias toward [[Leverage|leverage]] and clean workflows.

## What it unlocks for him

- One language across most of his product surface
- Fast iteration between local and deployed environments
- Access to a massive ecosystem of packages
- A foundation for the rest of his [[Working Stack|working stack]]

# Application Development

Application development is Duodode's engineering-heavy service line. It is the work of building real applications on top of [[Application Design]], using modern stacks and [[AI-Assisted Development]]. For [[Hendrix]], it is a natural extension of his belief that design and tech should ship together.

## Stack

- [[Next.js]] and [[Vercel]] for modern web applications
- [[Supabase]] for database, auth, and backend workflows
- [[Node.js]], [[GitHub]], [[VS Code]], [[Codex]], [[Claude Code]] for day-to-day engineering
- [[Postman]] for API testing
- [[n8n]] when automation threads through the app
- Motion via [[GSAP]], [[Framer Motion]], and [[Rive]]

## What it covers

- Front-end implementation of designed products
- Integration with databases, APIs, and external services
- AI-powered features embedded in the app
- Deployment, environments, and basic infrastructure hygiene

## The standard

Ship, then iterate. The work is judged by whether it runs in the real world, not by whether the repo looks elegant. That said, the code should be clean enough to scale and maintain. [[Taste]] applies to codebases the same way it applies to interfaces.

## Why it belongs in the agency

Without development, design stays theoretical. Duodode owns both so clients get a finished product, not a handoff. See [[Digital Agency]] and [[Digital Product Work]].
